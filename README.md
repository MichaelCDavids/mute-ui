# Mute-UI: A \"Muted\" React Component Library\n\n**A small, opinionated set of React components for building clean, consistent, and visually \"muted\" user interfaces with minimal effort.**\n\n[![npm version](https://badge.fury.io/js/%40michaelcdavids%2Fmute-ui.svg)](https://badge.fury.io/js/%40michaelcdavids%2Fmute-ui)\n\n## Live Demo\n\nA live demo of the component library is available at [https://mute-ui.web.app/](https://mute-ui.web.app/).\n\n## 🌟 Features\n\n*   **Theming:** A `ThemeProvider` at its core, allowing for easy customization of colors, spacing, and more.\n*   **Smart Components:** The `Box` component intelligently adapts its style based on its nesting level, creating a natural visual hierarchy.\n*   **Effortless Layout:** The `Stack` component simplifies layout and spacing, eliminating common CSS frustrations like margin collapse.\n*   **Rich Color Palette:** Includes a set of 18 beautiful, muted gradients.\n*   **Subtle Animations:** Add a touch of interactivity with pre-defined animations like \"pulse,\" \"shake,\" and \"jiggle.\"\n*   **Responsive by Design:** Components are designed to work seamlessly across different screen sizes.\n*   **Publicly Available:** Yes! `@MichaelCDavids/mute-ui` is a public package available on npm for you to use in your projects.\n\n## 🚀 Getting Started\n\n### Installation\n\nInstall the package using npm or your favorite package manager:\n\n```bash\nnpm install @MichaelCDavids/mute-ui\n```\n\n### Basic Usage\n\nWrap your application with the `ThemeProvider` and start building with `Box` and `Stack`.\n\n```tsx\nimport React from \'react\';\nimport ReactDOM from \'react-dom/client\';\nimport { ThemeProvider, Box, Stack } from \'@MichaelCDavids/mute-ui\';\n\nconst App = () => (\n    <ThemeProvider>\n        <Box padding=\"lg\">\n            <Stack gap=\"md\">\n                <h1>My Muted App</h1>\n                <p>Welcome to my new app!</p>\n                <Box as=\"button\" onClick={() => alert(\'Clicked!\')}>\n                    Click me\n                </Box>\n            </Stack>\n        </Box>\n    </ThemeProvider>\n);\n\nconst root = ReactDOM.createRoot(document.getElementById(\'root\'));\nroot.render(<App />);\n```\n\n## Advanced Usage\n\n### Custom Theming\n\nYou can easily override the default theme by passing a `customTheme` object to the `ThemeProvider`. This object is deep-merged with the default theme.\n\n```tsx\nimport { ThemeProvider, Box, Stack } from \'@MichaelCDavids/mute-ui\';\n\nconst customTheme = {\n  colors: {\n    primary: \'#6A0DAD\', // A vibrant purple\n    surface: \'#F3E5F5\',\n    inset: \'#E1BEE7\',\n    text: \'#311B92\',\n  },\n  spacing: {\n    xs: \'8px\',\n    sm: \'16px\',\n    md: \'24px\',\n    lg: \'32px\',\n    xl: \'40px\',\n  }\n};\n\nconst App = () => (\n    <ThemeProvider customTheme={customTheme}>\n        <Box>\n            <h1>My Custom Themed App</h1>\n        </Box>\n    </ThemeProvider>\n);\n```\n\n### Gradients and Animations\n\nMute-UI comes with a set of predefined gradients and animations. Here\'s how you might use them to create an animated button.\n\n```tsx\nimport { useTheme, Box } from \'@MichaelCDavids/mute-ui\';\nimport { useState } from \'react\';\n\nconst AnimatedButton = ({ gradient, animation, children }) => {\n  const theme = useTheme();\n  const [isAnimating, setIsAnimating] = useState(false);\n\n  const handleClick = () => {\n    setIsAnimating(true);\n    setTimeout(() => setIsAnimating(false), 500); // Animation duration\n  };\n\n  return (\n    <Box\n      as=\"button\"\n      onClick={handleClick}\n      style={{ \n        backgroundImage: theme.colors.gradients[gradient],\n        animation: isAnimating ? `${animation} 0.5s ease-in-out` : \'none\',\n      }}\n    >\n      {children}\n    </Box>\n  );\n};\n\n// Usage\n<AnimatedButton gradient=\"aurora\" animation=\"pulse\">\n  Pulse Me\n</AnimatedButton>\n```\n\n## Component API\n\n### `ThemeProvider`\n\nThe provider component that makes the theme available to the rest of your application.\n\n| Prop          | Type                  | Description                                        |\n|---------------|-----------------------|----------------------------------------------------|\n| `children`    | `React.ReactNode`     | The child components to render.                    |\n| `customTheme` | `DeepPartial<Theme>`  | A theme object to merge with the default theme.    |\n\n### `Box`\n\nThe fundamental, polymorphic building block.\n\n| Prop | Type | Description |\n|---|---|---|\n| `as` | `React.ElementType` | The HTML element to render. Defaults to `div`. |\n| `style` | `React.CSSProperties` | Standard React inline styles. |\n| `padding` | `keyof Theme[\'spacing\']` | The padding to apply, based on theme tokens. |\n| `elevation` | `0 \| 1 \| 2` | The shadow to apply, based on theme tokens. |\n| `...rest` | | All other props are passed to the underlying element. |\n\n### `Stack`\n\nA component for managing layout and spacing between elements.\n\n| Prop | Type | Description |\n|---|---|---|\n| `direction` | `\'row\' \| \'column\'` | The direction of the stack. Defaults to `\'column\'`. |\n| `gap` | `keyof Theme[\'spacing\']` | The space between elements, based on theme tokens. |\n| `style` | `React.CSSProperties` | Standard React inline styles. |\n\n\n## Theming\n\nThe theme object has the following structure. You can override any of these values using the `customTheme` prop on the `ThemeProvider`.\n\n```ts\nexport interface Theme {\n  colors: {\n    primary: string;\n    surface: string;\n    inset: string;\n    text: string;\n    accent: string;\n    gradients: {\n      primary: string;\n      secondary: string;\n      tertiary: string;\n      aurora: string;\n      dusk: string;\n      sand: string;\n      sky: string;\n      mist: string;\n      stone: string;\n      meadow: string;\n      rose: string;\n      peach: string;\n      ice: string;\n      lavender: string;\n      mint: string;\n      butter: string;\n      coral: string;\n      ocean: string;\n    };\n  };\n  spacing: {\n    xs: string;\n    sm: string;\n    md: string;\n    lg: string;\n    xl: string;\n  };\n  shadows: {\n    0: string;\n    1: string;\n    2: string;\n  };\n}\n```\n\n## Versioning\n\nThis project adheres to [Semantic Versioning](https.semver.org/).
